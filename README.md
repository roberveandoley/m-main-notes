# m-main-notes

## 项目简介

`m-main-notes` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常用于记录、展示或测试某些前端功能或内容片段，不针对任何特定网站或域名。

## 目录说明

```
m-main-notes/
├── index.html          # 仓库主页（可选）
├── page-1.html         # 示例页面
├── page-2.html         # 示例页面
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- **根目录**：存放各独立 HTML 页面文件，命名建议简洁且能体现页面内容。
- **assets/**：存放页面共享的样式、脚本、图片等资源文件。如果页面使用独立资源，建议在页面所在目录下自建文件夹管理。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，页面之间无强制关联。每个页面可以：

- 包含完整的 HTML、CSS 和 JavaScript。
- 引用仓库内 `assets/` 下的公共资源。
- 引用外部 CDN 或第三方库（需确保稳定性与合规性）。

**页面命名建议**：使用英文或拼音，避免特殊字符，便于直接通过浏览器访问。

## 维护说明

- **新增页面**：直接将 HTML 文件添加到根目录或合适子目录，并更新 `index.html`（如有）中的链接。
- **更新页面**：直接修改对应文件，保留历史版本可通过 Git 记录回溯。
- **删除页面**：移除文件及其关联资源，避免遗留无效引用。
- **资源管理**：公共资源统一放入 `assets/`，页面私有资源建议放在页面同级目录下。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/m-main-notes.git
   ```
2. 直接在浏览器中打开任意 `.html` 文件即可查看对应页面。
3. 若需部署，可将仓库内容上传至任意静态托管服务（如 GitHub Pages、Netlify 等），根目录下的 HTML 文件将自动可访问。
